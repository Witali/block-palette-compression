# Git commits

- When working in this Git repository and changing files, commit each logically complete change as a separate, focused commit.
- Stage only files that belong to that completed change. Preserve unrelated user changes and generated or temporary files.
- Use a concise commit message that describes the completed change.

# Git worktrees

- Create new linked Git worktrees under the repository's `.tmp/` directory.
- Use a descriptive subdirectory name for each worktree and keep `.tmp/` ignored by Git.

# Documentation

- Write and save project documentation in English by default.
- If a Russian version is needed, keep it in a separate file with the `_ru` suffix before the file extension.
