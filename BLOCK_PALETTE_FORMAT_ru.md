# BPAL v4

[English version](./BLOCK_PALETTE_FORMAT.md)

`BPAL` — компактный формат изображений с блочными палитрами. Все многобитные
значения записываются от старшего бита к младшему. Версия 4 поддерживает 1, 2,
4 или 8 явных общих палитр. Каждый блок выбирает одну общую палитру и хранит
небольшую таблицу индексов, локальных для этой палитры.

Декодер сохраняет совместимость с BPAL v1-v3 и устаревшей векторной моделью
палитры. Кодировщик v4 всегда записывает явные палитры.

## Заголовок v4

Первые 4 байта — ASCII-сигнатура `BPAL`. За ней следует 80-битное поле:

| Смещение, бит | Размер | Значение |
| ---: | ---: | --- |
| 0 | 4 | Версия формата; `4` для v4 |
| 4 | 24 | Ширина минус 1 |
| 28 | 24 | Высота минус 1 |
| 52 | 3 | `log2(размер блока) - 1` |
| 55 | 2 | `log2(цветов в блоке) - 1` |
| 57 | 4 | `log2(цветов в каждой общей палитре) - 1` |
| 61 | 1 | Формат цвета: `0` — RGB565, `1` — RGB888 |
| 62 | 1 | Модель палитры: `0` — явная, `1` — устаревшая векторная |
| 63 | 9 | В новых файлах ноль; число векторов минус 1 для старой модели |
| 72 | 1 | В новых файлах ноль; пространство старой модели: `0` — RGB, `1` — OKLab |
| 73 | 2 | `log2(числа общих палитр)`: `0`, `1`, `2` или `3` |
| 75 | 5 | Зарезервировано; ноль в v4 |

Полный служебный заголовок v4 занимает 14 байт: 4 байта сигнатуры и 10 байт
битовых полей.

## Полезная нагрузка

Пусть `P` — число общих палитр, `G` — число цветов в каждой общей палитре,
а `L` — число локальных цветов в каждом блоке. Сразу после заголовка без
промежуточного выравнивания записываются:

1. `P × G` цветов, сгруппированных по номеру общей палитры. Цвет занимает 16
   или 24 бита. В старых векторных файлах вместо этого записаны начальный и
   конечный цвета каждого вектора.
2. Селектор общей палитры каждого блока в построчном порядке блоков. Селектор
   занимает `log2(P)` бит; при `P = 1` этот раздел имеет нулевую длину.
3. По `L` локальных для выбранной палитры индексов каждого блока. Один индекс
   занимает `log2(G)` бит.
4. Локальный индекс таблицы блока каждого пикселя в построчном порядке. Один
   индекс занимает `log2(L)` бит.

Для блока `b` и локальной позиции пикселя `i` итоговый индекс в плоском массиве
общих палитр вычисляется так:

```text
blockSelector[b] × G + blockPalette[b × L + i]
```

Если `B` — число блоков, `N` — число пикселей, а `C` — число бит на сохранённый
цвет, длина полезной нагрузки до финального выравнивания равна:

```text
P × G × C + B × log2(P) + B × L × log2(G) + N × log2(L)
```

При необходимости нулевыми битами дополняется только последний байт файла.
Альфа-канал не хранится; декодированное изображение считается непрозрачным.

## Совместимость со старыми версиями

Файлы BPAL v1-v3 неявно содержат одну общую палитру, поэтому ширина селектора
равна нулю. Заголовок v1 после сигнатуры занимает 64 бита. Заголовки v2 и v3
занимают 80 бит; v3 добавляет бит цветового пространства в позиции 72.

Цвета старой векторной палитры восстанавливаются линейной интерполяцией в RGB
или OKLab, указанном в заголовке. Векторная модель допустима только для одной
общей палитры.
